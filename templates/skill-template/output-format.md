## Formatting Rules

### General
- Use Markdown format
- Include section headers
- Use consistent formatting
- Add placeholder citations [1], [2], etc.

### Language
- Use academic tone
- Be concise and clear
- Avoid jargon (or explain it)
- Use active voice when possible

### Structure
- Follow logical flow
- Use numbered lists for steps
- Use bullet points for items
- Include code blocks for code

---

## Quality Indicators

### Good Output
✅ Clear structure
✅ Complete sections
✅ Specific details
✅ Actionable content
✅ Proper formatting

### Bad Output
❌ Vague descriptions
❌ Missing sections
❌ Generic content
❌ No actionable items
❌ Poor formatting
